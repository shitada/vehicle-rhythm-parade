# のりものリズムパレード — 仕様書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| タイトル | のりものリズムパレード |
| 対象年齢 | 4〜5歳向け |
| ジャンル | リズムタップゲーム |
| 技術構成 | HTML / CSS / JavaScript（フレームワークなし、単一ページ） |
| フォント | Zen Maru Gothic（Google Fonts） |
| 音声 | Web Audio API によるリアルタイム音声合成（音声ファイル不使用） |

### コンセプト
画面上で光るオーブ（パルス）に合わせてタップボタンを押す、幼児向けリズムゲーム。乗り物をテーマにしたラウンドを順番にクリアしていく。

## 3. 画面構成

アプリは 3 つの画面（`<section class="screen">`）で構成され、`screen-active` クラスの切り替えで画面遷移する。

### 3.1 タイトル画面（`data-screen="title"`）

- **ヒーローカード**: タイトル「のりもの リズムパレード」、説明文「ひかったら タップ。おとと いっしょに あそぼう。」
- **「あそぶ」ボタン**: ゲーム開始。カード全体のクリック / タップ / Enter / Space キーでも開始可能。
- **乗り物バッジ一覧**: 10 種の乗り物名をカラーバッジで表示（でんしゃ、バス、しょうぼうしゃ、タクシー、ふね、ヘリ、パトカー、ブルドーザー、ひこうき、ロケット）。

### 3.2 ゲーム画面（`data-screen="game"`）

- **ステータスバー**: ラウンド表示（例: `ラウンド 1 / 3`）、プログレスドット、ホームボタン
- **リワードボード**: スター数（⭐ ◯こ）、連続ヒット数（🎉 ◯れんぞく）、フィーバーサイン
- **プレイフィールド**:
  - 雲デコレーション（左右）
  - 乗り物ステージ（絵文字・名前・サブタイトル・操作説明・ヒントアイコン）
  - パルスレーン（光るオーブが並ぶエリア）
  - タップボタン（画面下部の大きなボタン）
  - 応援バブル（タップ時のフィードバックテキスト）
  - ラウンドバナー（ラウンド開始・クリア時の通知）
  - リワードバースト（スター獲得時のエフェクト）
  - 休憩ストップ（ラウンド間の休憩画面、バックドロップブラー付き）
  - サプライズポップ（特定パルスで出現する演出）

### 3.3 リザルト画面（`data-screen="result"`）

- **リザルトカード**: クリアタイトル（例: `10だい クリア`）、合計タップ数の報告、ボーナス情報（スター数・最高連続数・サプライズ回数）
- **「もういちど」ボタン**: ゲーム再開。カード全体のクリック / キーでも可。
- **ホームボタン**: タイトル画面に戻る。

## 4. リズムパターンシステム

### 4.1 概要

各ラウンドには乗り物固有の **リズムパターン** が定義されている。パターンは繰り返し（ループ）再生され、プレイヤーはリズムに乗ってタップする。等間隔ではなく、乗り物ごとに異なるリズム感が特徴。

### 4.2 ビートタイプ

パターンは 3 種類のビートで構成される:

| ビート | コード | アクティブ時の表示 | プレイヤーの行動 |
|--------|--------|-------------------|----------------|
| **タップ** | `1` | 金色に光る ✨ | タップする |
| **おやすみ** | `-1` | 赤色に光る 💤 | タップしない（我慢する） |
| **ポーズ** | `0` | オーブなし（休符） | リズムの間（ま） |

### 4.3 パターンの構成

- `rhythm` 配列: ビートコードの配列（例: `[1, 1, 0, 0, 1, 1, 0, 0]`）
- `beatInterval`: 1 ビートスロットあたりの時間（ms）
- `loops`: パターンの繰り返し回数
- パルスレーンには**タップ + おやすみオーブのみ表示**（ポーズ = 0 は非表示）
- **全オーブは非アクティブ時は同じ見た目**（光るまでタップ/おやすみの区別は不可能）
- ビートシーケンス構築: `rhythm × loops` で全ビート配列を生成し、非ゼロビートのみ抽出して orbSequence を構築

### 4.4 おやすみオーブ

- ラウンド 1〜4 にはおやすみオーブなし（リズムパターンのみで練習）
- ラウンド 5 で初登場し、以降徐々に増加
- 非アクティブ時は通常オーブと同じ見た目 → 光った瞬間に金色/赤色を判断する必要がある
- おやすみオーブをタップした場合: ストリーク（連続ヒット）リセット、ブッ音再生、ボタン赤フラッシュ（ラウンド失敗にはならない）
- おやすみオーブを正しく見送った場合: ペナルティなし（オーブは灰色に変化）

### 5.4 ボーナスステージ（ミックスラウンド）

既存 10 ラウンド完了後に登場するボーナスステージ。5 ラウンド。
各ラウンドでは複数の乗り物のリズムパターンがランダムに切り替わる。

| # | ID | 名前 | 絵文字 | テーマカラー | 組み合わせ元 | Beat(ms) | Loops | 必要ヒット |
|---|-----|------|--------|-------------|----------|----------|-------|----------|
| 11 | mix-city | まちの のりもの | 🏙️ | #ff6b9d | でんしゃ+バス+タクシー | 450 | 8 | 14 |
| 12 | mix-work | はたらく くるま | 🔧 | #e8590c | しょうぼうしゃ+パトカー+ブルドーザー | 400 | 8 | 15 |
| 13 | mix-sky | そらの のりもの | 🌤️ | #1098ad | ヘリ+ひこうき+ロケット | 350 | 8 | 14 |
| 14 | mix-allstar | オールスター | 🌟 | #ae3ec9 | 全10種からランダム | 380 | 10 | 18 |
| 15 | mix-final | ファイナル パレード | 🎪 | #d6336c | 全10種からランダム | 320 | 12 | 22 |

**ミックスラウンドの仕組み:**

- 各ラウンドは `isMix: true` フラグと `mixSources` 配列（使用する乗り物ID）を持つ
- `buildMixedBeatSequence(round)` が各ループごとにランダムに 1 つの乗り物を選択し、その rhythm を使用
- 乗り物が切り替わるタイミングでステージの絵文字・名前・サブタイトルがアニメーションで更新される
- 音は切替先の乗り物の固有音（pulseFrequency / pulseType）を使用

### 5.4 ラウンドごとの固有プロパティ

各ラウンドは以下のプロパティを持つ:

- `rhythm` — リズムパターン配列（`1` = タップ、`-1` = おやすみ、`0` = ポーズ）
- `beatInterval` — 1 ビートスロットの時間（ms）
- `loops` — パターンの繰り返し回数
- `hitsNeeded` — クリアに必要な最低ヒット数
- `subtitle` — サブタイトル（擬音語: ガタン ゴトン、ゆら ゆら 等）
- `instruction` — タップ操作の説明テキスト
- `themeClass` — CSS テーマクラス名（プレイフィールドの配色変更）
- `bubbleText` — タップ成功時のバブルテキスト
- `introText` — ラウンド開始時のバナーテキスト
- `surpriseTitle` / `surpriseCopy` / `surpriseIcons` — サプライズ演出の内容
- `surpriseMoments` — サプライズが発動するオーブインデックス（配列）
- `pulseType` — パルス音のオシレータ波形（sine / triangle / square / sawtooth）
- `pulseFrequency` — パルス音の基本周波数（Hz）
- `hitTone` — ヒット音の周波数（Hz）
- `bgmNotes` — BGM のノート配列（4音のループ）
- `bgmType` — BGM のオシレータ波形
- `tapPraise` / `starPraise` / `clearPraise` / `feverPraise` — 各場面での称賛テキスト

ミックスラウンドは追加で以下を持つ:

- `isMix` — `true` が設定される
- `mixSources` — 使用する乗り物 ID の配列

## 6. ゲームメカニクス

### 6.1 基本フロー

1. タイトル画面で「あそぶ」を押す → AudioContext 初期化
2. ラウンド準備（`prepareRound`）: UI 更新、ビートシーケンス構築、パルスオーブ生成、BGM 開始
3. 1200ms 後におてほんフェーズ開始（`runPreview`）
4. おてほん: 1 ループ分のリズムをオーブ + 音で自動再生（タップ不要・判定なし）
5. おてほん完了 → バナー「さあ はじめよう！」→ 1200ms 後に本番シーケンス開始
6. リズムパターンに従いオーブが順番に光る → プレイヤーが判断してタップ/見送り
7. 全ビート終了後 → ラウンドクリア処理（`finishRound`）
8. 休憩画面表示 → 1800ms 後に次のラウンド準備
9. 全ラウンド終了 → リザルト画面表示

**ミックスラウンドの場合:** おてほんの代わりにバナー「いろんな のりものが でるよ！」を表示して本番開始

### 6.2 おてほんフェーズ

`runPreview(round, callback)` 関数:

1. 1 ループ分のリズムパターンのオーブを生成
2. タップボタンを「👀 みてね」に変更し半透明化（`preview-mode` クラス）
3. オーブが自動で光り、音が鳴る（タップオーブは金色、おやすみは赤）
4. `state.previewMode = true` でタップ判定を無効化
5. 完了後: ボタンを「👆 タップ」に戻し、callback を実行

### 6.3 ビートシーケンス構築

`buildBeatSequence(round)` 関数:

1. `round.rhythm` を `round.loops` 回繰り返して全ビート配列を生成
2. 配列を走査し、ビート値が 0 でない要素のみ抽出
3. 各要素に `orbIndex`（表示オーブの通し番号）、`beatTime`（開始からの経過ms）、`type`（`'tap'` or `'rest'`）を付与
4. 結果の orbSequence 配列を返す

### 6.4 タップ判定

5 種類の判定:

| 状態 | タップ結果 | 演出 |
|------|-----------|------|
| **おてほん中** | 無視 | バブル「👀 みてるよ！」 |
| **先行判定窓内（タップオーブ）** | ヒット ✅ | オーブ即座に点灯→緑変化、称賛テキスト、ヒット音 |
| **タップオーブがアクティブ** | ヒット ✅ | オーブ緑変化、称賛テキスト、ヒット音 |
| **おやすみオーブがアクティブ（または先行窓内）** | ミス ❌ | ストリークリセット、ブッ音、「💤 おやすみちゅう！」、ボタン赤フラッシュ |
| **非アクティブ中** | ミス ❌ | ストリークリセット、「ナイスタップ」、低い音 |

**タイミング判定窓:**

```
beatInterval (例: 500ms)
|←─────────────────────────→|
     ┃━━━━━━━━━━━━━━━━━━━━━┃
     ↑                      ↑ 判定終了 (75%)
├早い┤├── 通常ヒット ──┤
先行(20%)   点灯後(75%)
```

- **先行判定窓**: オーブ点灯の `beatInterval × 20%` 前から受付開始
- **通常判定窓**: オーブ点灯から `beatInterval × 75%` 後まで
- **合計受付窓**: `beatInterval × 95%`（以前は 75%）
- 先行ヒット時: オーブが即座に点灯→緑に変化（フィードバック感維持）
- おやすみオーブの先行窓でタップした場合もミス判定
- 休憩中のタップ: 「つぎの じゅんびちゅう」表示

### 6.5 リワードシステム

| 機能 | 条件 | 演出 |
|------|------|------|
| **スター獲得** | 3 ヒット連続ごと（streak % 3 === 0） | スター数 +1、スター称賛テキスト、リワードバースト、追加音 |
| **フィーバーモード** | 5 ヒット連続ごと（streak % 5 === 0） | プレイフィールドに光るボーダー、フィーバーサイン活性化、900ms 持続 |
| **連続記録** | ヒットするたびに加算、ミス/おやすみタップでリセット | 最高連続数はリザルトに表示 |

### 6.6 サプライズ演出

- 各ラウンドの `surpriseMoments` で指定されたオーブインデックスに到達するとサプライズカードが表示
- 表示内容: ラウンド固有のアイコン・タイトル・説明文
- 900ms で自動的に非表示
- 同じオーブでの重複発動は `triggeredSurprises` 配列で防止

## 7. 音声システム

Web Audio API を使用し、全ての音声をリアルタイム生成する（外部音声ファイル不使用）。

### 7.1 音量設定

- マスターゲイン: `0.18`
- BGM ゲイン: マスター × `0.28`（= 約 `0.05`）

### 7.2 音声の種類

| 音声 | トリガー | 詳細 |
|------|----------|------|
| **パルス音** | タップオーブ点灯時 | ラウンド固有の波形・周波数。オーブ番号に応じて +18Hz ずつ上昇 |
| **おやすみ音** | おやすみオーブ点灯時 | 低い警告音（パルス周波数 × 0.6）、同じ波形 |
| **ヒット音** | タップ成功時 | ラウンド固有のヒット音周波数 |
| **おやすみタップ音** | おやすみオーブタップ時 | 220Hz、square 波、0.12s（ブッ音） |
| **スター獲得音** | 3 連続ヒット時 | ヒット音 +80Hz、triangle 波 |
| **フィーバー音** | 5 連続ヒット時 | ヒット音 +120Hz、triangle 波 |
| **サプライズ音** | サプライズ発動時 | ヒット音 +170Hz、triangle 波 |
| **ラウンドクリア音** | ラウンド終了時 | 660Hz、triangle 波 |
| **ゲームクリア音** | 全ラウンド終了時 | 523Hz → 659Hz → 784Hz の 3 音シーケンス |
| **ミスタップ音** | 非アクティブ時タップ | 420Hz、sine 波、短め |
| **BGM** | ラウンド中常時 | ラウンド固有の 4 音ループ + オクターブ下のハーモニー |

### 7.3 BGM

- 4 音のノート配列をループ再生
- メインメロディ + 1 オクターブ下のハーモニー（ゲイン半分）
- テンポ: `round.beatInterval × 0.42`（最低 360ms）

## 8. 状態管理

ゲーム状態は `state` オブジェクトで管理される。`createInitialState()` で初期化。

```javascript
{
  roundIndex: 0,        // 現在のラウンドインデックス
  activePulse: -1,      // 現在アクティブなオーブのインデックス
  activeOrbType: null,   // 現在アクティブなオーブの種類 ('tap' | 'rest' | null)
  hits: 0,              // 現在のラウンドでのヒット数
  totalHits: 0,         // 全ラウンド合計ヒット数
  acceptingTap: false,   // タップ受付中フラグ
  timers: [],           // setTimeout の ID 配列（クリーンアップ用）
  finishedRounds: 0,    // クリアしたラウンド数
  roundResults: [],     // 各ラウンドの結果配列
  stars: 0,             // 獲得スター数
  streak: 0,            // 現在の連続ヒット数
  bestStreak: 0,        // 最高連続ヒット数
  surprises: 0,         // サプライズ発動回数
  triggeredSurprises: [], // 発動済みサプライズのキー配列
  feverTimer: null,     // フィーバーモードの解除タイマー
  restMisses: 0,        // おやすみオーブをタップした回数
  previewMode: false,   // おてほんフェーズ中フラグ
  currentMixSequence: null, // ミックスラウンドの事前生成済みシーケンス
}
```

## 9. UI / UX 仕様

### 8.1 レイアウト

- 中央配置のシングルカラムレイアウト（最大幅 900px）
- カード型コンポーネント（角丸 36px、半透明白背景、ドロップシャドウ）
- Zen Maru Gothic フォント（丸みのある読みやすい書体）

### 8.2 パルスレーンの2列表示

- パルスレーンは `flex-wrap: wrap` で折り返しを許可
- R1〜R6（オーブ数 12〜20）: 1列に収まる
- R7〜R10（オーブ数 30）: 自動的に2列に折り返して表示
- ボーナスラウンド（ソース依存で多数オーブ）: 同様に2列表示
- 行間に `row-gap: 12px` を設定し、視覚的に区別しやすくする
- プレイ順序は左上から右下へ（1列目が先に光り、2列目へ進む）

### 8.3 テーマカラー

- プレイフィールドの配色はラウンドごとの `themeClass` で切り替え
- CSS カスタムプロパティ `--round-accent` / `--round-accent-soft` で制御
- 各テーマに対応する背景グラデーション・タップボタン色・パルスオーブの光り色

### 8.3 アニメーション

| アニメーション | 対象 | 内容 |
|--------------|------|------|
| `pulse-glow` | タップボタン | タップ時の拡縮（420ms） |
| `bubble-up` | 応援バブル | 下から上にフェードアウト（900ms） |
| `banner-drop` | ラウンドバナー | 上からドロップイン → フェードアウト（1400ms） |
| フィーバーモード | プレイフィールド | 金色のボックスシャドウ追加 |

### 8.4 レスポンシブ対応

- 640px 以下で専用スタイル適用
  - パディング縮小、角丸縮小（28px）
  - プレイフィールド最小高さ 82vh に拡大
  - パルスレーンのギャップ縮小
  - リワードボードのピルを 100% 幅に
  - サプライズポップの位置調整

### 8.5 アクセシビリティ

- 装飾要素に `aria-hidden="true"`
- ボタンに `aria-label` 設定
- カード要素に `role="button"` / `tabindex="0"` / `aria-label`
- フィーバーサインに `aria-live="polite"`
- プログレスドットに `aria-label="しんこう"`
- キーボード操作対応（Enter / Space キー）

## 10. 操作方法

| 操作 | 画面 | 動作 |
|------|------|------|
| 「あそぶ」ボタン / カードクリック / Space / Enter | タイトル | ゲーム開始 |
| タップボタン / Space | ゲーム | パルスにタイミングを合わせてタップ |
| 🏠 ボタン | ゲーム / リザルト | タイトル画面に戻る |
| 「もういちど」ボタン / カードクリック / Space / Enter | リザルト | ゲーム再開 |

## 10. 技術的な補足

- **タイマー管理**: 全ての `setTimeout` / `setInterval` の ID を `state.timers` に保存し、画面遷移時に `clearTimers()` で一括クリア
- **AudioContext**: ユーザー操作時に初期化（ブラウザの自動再生ポリシー対応）。サスペンド時は `resume()` で復帰
- **DOM 操作**: `querySelector` でキャッシュした要素参照を使用。パルスオーブは毎ラウンド再生成
- **touch-action: manipulation**: タップ遅延を排除し、ダブルタップズームを防止
